# 전용 라우터(공유기) + PC 서버 + 안드로이드 가이드

**강력 추천하는 구성입니다.**
가이드 폰을 핫스팟으로 사용하면 **발열과 배터리 소모**가 심해져 방송이 끊길 위험이 있습니다.
별도의 **와이파이 공유기(Router)**를 사용하면 트래픽 처리 부담을 공유기가 가져가므로 시스템이 훨씬 안정적입니다.

## 1. 연결 구조

*   **공유기 (Router)**: 중앙에서 와이파이 신호를 쏘고 중계를 담당합니다.
*   **PC (서버)**: 공유기에 **유선 랜선**으로 연결하는 것을 추천합니다. (무선도 가능하지만 유선이 더 안정적)
*   **가이드 폰 & 관광객 폰**: 공유기의 **와이파이**에 접속합니다.

## 2. 공유기 설치 및 연결

1.  **공유기 전원 켜기**: 행사 장소에 공유기를 설치하고 전원을 켭니다. (인터넷 선이 없어도 내부 통신은 가능합니다.)
2.  **PC 연결**:
    *   (추천) 랜선으로 PC와 공유기의 **LAN 포트**를 연결합니다.
    *   (또는) PC 와이파이를 켜서 공유기 와이파이에 연결합니다.
3.  **핸드폰 연결**:
    *   가이드 폰과 관광객 폰 모두 **공유기 와이파이**에 연결합니다.
    *   **모바일 데이터**는 끄는 것이 좋습니다.

## 3. PC 서버 실행 및 IP 확인

공유기에 연결되었으므로 PC의 IP 주소가 변경됩니다.

1.  PC 터미널에서 `ipconfig` (Mac은 `ifconfig`)를 입력합니다.
2.  새로운 **IPv4 주소**를 확인합니다. (예: `192.168.0.100`)
3.  서버를 켭니다.
    ```bash
    python server.py
    ```

## 4. 가이드 폰 설정 업데이트 (중요!)

IP 주소가 바뀌었으므로 가이드 폰의 마이크 권한 설정을 다시 해야 합니다.

1.  가이드 폰 크롬 브라우저 실행.
2.  주소창에 `chrome://flags` 입력.
3.  `Insecure origins treated as secure` 항목에서 기존 주소를 지웁니다.
4.  방금 확인한 새로운 주소(예: `http://192.168.0.100:5000`)를 입력합니다.
5.  **Relaunch** 버튼을 눌러 재시작합니다.

## 5. 접속

이제 모든 기기(가이드/관광객)에서 새 주소로 접속하면 됩니다.

*   주소: `http://[PC의_새로운_IP]:5000`

---

## 팁: 인터넷이 없는 공유기라면?
인터넷 선(WAN)을 꽂지 않은 "깡통 공유기" 상태라도 **내부 네트워크(인트라넷)**는 정상 작동합니다.
*   핸드폰에서 "인터넷 연결 확실치 않음" 경고가 떠도 **"연결 유지"**를 선택하면 됩니다.
*   이 상태에서는 앱 다운로드나 카톡은 안 되지만, **가이드 방송은 완벽하게 작동**합니다.
