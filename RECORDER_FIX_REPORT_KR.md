# 레코더 모드 오디오 전송 오류 수정 보고서

## 문제 상황
사용자가 **[🔊 Recorder Mode]**를 선택했을 때, 오디오가 제대로 들리지 않거나 전송되지 않는 문제가 발생했습니다. (로그 상에서 1바이트 데이터만 계속 생성됨)

## 원인 분석
안드로이드의 모바일 브라우저(특히 Chrome/Samsung Internet)에서 `MediaRecorder`를 사용할 때, 데이터를 잘라내는 간격(timeslice)을 `50ms`로 너무 짧게 설정하면 오디오 인코딩이 제대로 이루어지지 않고 빈 헤더 데이터(1바이트)만 생성되는 문제가 있었습니다.

## 해결 방법
1.  **전송 간격 조정:** `MediaRecorder`의 데이터 수집 간격을 기존 `50ms`에서 **`250ms`**로 늘렸습니다. 약간의 지연(0.2초)이 생기지만, 오디오 데이터가 안정적으로 생성되어 끊김 없이 들리게 됩니다.
2.  **빈 데이터 필터링:** 10바이트 미만의 의미 없는 데이터는 전송하지 않도록 필터링 코드를 추가하여 네트워크 낭비를 줄였습니다.

## 적용 결과
- 이제 레코더 모드에서도 목소리가 정상적으로 전달될 것입니다.
- 수정된 코드를 GitHub에 푸시 완료했습니다.
