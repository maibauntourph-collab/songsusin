# 🚀 60명 동시 접속 가능여부 및 가이드

"60명까지 가능할까?"에 대한 답변입니다.
**가능합니다. 하지만, 일반적인 스마트폰 핫스팟으로는 불가능하며 '공유기'가 필수입니다.**

## 1. 네트워크 한계 (가장 중요)

스마트폰의 '모바일 핫스팟' 기능은 하드웨어적으로 접속 기기 수를 제한합니다.

| 방식 | 최대 접속자 (대략) | 안정성 | 비고 |
|:---:|:---:|:---:|:---|
| **스마트폰 핫스팟** | **8 ~ 10명** | 낮음 | 대부분의 안드로이드 폰은 10명 제한이 걸려있습니다. |
| **태블릿 핫스팟** | **10 ~ 32명** | 보통 | 고성능 태블릿은 더 많이 허용하기도 하지만, 60명은 무리입니다. |
| **무선 공유기 (Router)** | **50 ~ 100명+** | **높음** | **60명을 위해선 필수입니다.** |

### ✅ 해결책: 휴대용 무선 공유기 사용
60명의 관광객을 수용하려면 **ipTIME, TP-Link** 등의 휴대용/여행용 무선 공유기가 반드시 필요합니다.

1.  **공유기 전원 켜기**: 휴대용 배터리로 구동 가능한 모델 추천.
2.  **서버(태블릿/폰) 연결**: 공유기 와이파이에 접속.
3.  **관광객 연결**: 모두 공유기 와이파이에 접속.
4.  **서버 실행**: 태블릿에서 `python server.py` 실행 후 할당받은 IP (예: `192.168.0.x`) 확인.
5.  **접속**: 관광객들은 브라우저에 해당 IP를 입력하여 접속.

## 2. 서버 성능 (CPU 부하)

이 시스템(Python `aiortc`)은 각 사용자마다 오디오를 암호화해서 전송합니다.
*   **스마트폰(Termux) 서버**: 약 **10~20명** 정도는 무난하지만, 60명의 암호화 처리는 발열과 버벅임을 유발할 수 있습니다.
*   **자동 전환 모드**: 시스템에는 WebRTC가 느려지면 더 가벼운 방식(WebSocket)으로 자동 전환하는 기능이 있어 어느 정도 버틸 수 있습니다.

### 💡 추천 구성 (60명 기준)
*   **Best**: **노트북(Laptop)** (서버) + **무선 공유기** (네트워크)
*   **Good**: **최신 고성능 태블릿** (서버) + **무선 공유기** (네트워크)
*   **Bad**: 스마트폰 핫스팟 하나로 해결하려고 하는 것 (10명 이상 접속 불가)

## 3. 결론
*   **네트워크**: 폰 핫스팟으로는 60명 접속 자체가 안 됩니다. **공유기**를 챙기세요.
*   **서버 기기**: 폰보다는 **태블릿**이나 **노트북**을 서버로 쓰세요.
*   **배터리**: 60명의 데이터를 처리하면 서버 기기 배터리가 빨리 닳습니다. 보조배터리를 꼭 연결하세요.
