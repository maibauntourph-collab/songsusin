# 안드로이드 STT 종료 버그 수정 보고서

## 문제 상황
사용자가 **[🔊 Recorder Mode]** (오디오 전용 모드)를 선택했음에도 불구하고, 내부적으로 STT(음성 인식) 엔진이 계속 재시작되면서 "띠링" 소리가 반복되는 현상이 발생했습니다.

## 원인 분석
`static/app.js`의 `recognition.onend` (음성 인식 종료 시 자동 재시작 로직)에서 현재 모드가 `stt`인지 확인하는 조건이 빠져 있었습니다. 이로 인해 방송 중(`isBroadcasting == true`)이기만 하면 무조건 STT를 다시 켜버리는 문제가 있었습니다.

## 해결 방법
1.  **재시작 조건 강화:** `recognition.onend` 이벤트 핸들러에서 재시작 조건을 `if (isBroadcasting && audioMode === 'stt')`로 수정하여, 오직 STT 모드일 때만 재시작하도록 변경했습니다.
2.  **모드 변경 시 강제 종료:** `setAudioMode` 함수에서 'recorder' 모드로 변경 시, 현재 실행 중인 STT 엔진이 있다면 즉시 `stop()` 하도록 코드를 추가했습니다.

## 적용 결과
- 이제 **Recorder Mode**를 선택하면 STT 엔진이 완전히 종료되며, 더 이상 "띠링" 소리가 나지 않습니다.
- 수정된 코드를 GitHub에 푸시 완료했습니다.
