# 📱 태블릿을 메인 서버로 활용하기 (Tablet as Dedicated Server)

스마트폰 대신 **태블릿(갤럭시 탭, 안드로이드 패드 등)**을 서버로 사용하면 더욱 안정적인 가이드 시스템 운영이 가능합니다.

## 1. 태블릿 서버의 장점
1. **대용량 배터리**: 스마트폰보다 배터리 용량이 훨씬 커서 장시간(투어 내내) 서버를 켜두기에 유리합니다.
2. **분리형 운용**: 태블릿은 가방에 넣어두고(서버+핫스팟 역할), 가이드는 가벼운 본인 폰만 들고(마이크 역할) 진행할 수 있습니다.
3. **발열 분산**: 서버 구동과 통신 중계 부하를 태블릿이 담당하므로, 가이드 폰의 발열과 배터리 소모가 줄어듭니다.

## 2. 구성도 (추천 시나리오)

*   **태블릿 (서버 + 공유기)**:
    *   **역할**: Wi-Fi 핫스팟 켜기, `server.py` 실행.
    *   **위치**: 가이드의 가방 속 또는 안전한 곳.
*   **가이드 폰 (입력 장치)**:
    *   **역할**: 태블릿 와이파이 접속 -> 브라우저로 서버 주소 접속 -> **"I am Guide"** 선택.
    *   **장점**: 폰이 가벼워지고 화면을 계속 켜둘 필요가 없음 (백그라운드 전송).
*   **관광객 폰 (수신 장치)**:
    *   **역할**: 태블릿 와이파이 접속 -> **"I am Tourist"** 선택.

## 3. 안드로이드 태블릿 설정 (Galaxy Tab, Lenovo Pad 등)

설정 방법은 안드로이드 스마트폰과 **100% 동일**합니다. [MOBILE_GUIDE.md](./MOBILE_GUIDE.md) 문서를 참고하세요.

### 요약:
1.  **Termux** 앱 설치 (Play Store 또는 F-Droid).
2.  `git clone ...` 으로 코드 다운로드.
3.  `./termux_install.sh` 실행.
4.  **모바일 핫스팟** 켜기 (비밀번호 설정 권장).
5.  `python server.py` 실행.
6.  `ifconfig` 명령어로 태블릿의 IP 주소 확인 (예: `192.168.43.1`).

## 4. 아이패드 (iPad) 사용 불가 안내
*   **서버 구동용**: 아이패드는 iOS 정책상 백그라운드 프로세스 유지와 외부 포트 개방이 매우 제한적이라 **메인 서버로 사용하기 부적합**합니다.
*   **클라이언트용**: 아이패드를 **가이드용 화면**이나 **손님용**으로 사용하는 것은 완벽하게 지원됩니다.

## 5. 팁 (Tip)
*   태블릿을 서버로 쓸 때는 **화면 자동 꺼짐 시간**을 길게 설정하거나, `Termux` 알림 설정에서 **"Acquire Wakelock"**을 켜두는 것이 좋습니다. (화면이 꺼져도 CPU가 계속 돌도록)
