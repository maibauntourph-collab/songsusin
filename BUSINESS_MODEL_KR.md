# 모바일 오디오 가이드 시스템 - 비즈니스 모델 요약

## 출원 목적 문서
**작성일**: 2026년 1월 8일

---

## 1. 서비스 개요

**명칭**: 실시간 다국어 오디오 가이드 시스템

**핵심 기능**: 투어 가이드가 한국어로 설명하면, 관광객은 실시간으로 100개 이상의 언어로 번역된 자막과 함께 오디오를 수신하는 모바일 기반 서비스

**서비스 형태**: 웹 기반 SaaS (앱 설치 불필요)

---

## 2. 기술적 특징

### 핵심 기술
| 구분 | 기술 | 설명 |
|------|------|------|
| 음성 스트리밍 | WebRTC + WebSocket | 하이브리드 실시간 전송 |
| 음성인식(STT) | Web Speech API | 실시간 음성→텍스트 변환 |
| 번역 | Google Translate API | 100+ 언어 동시 지원 |
| 동기화 | Socket.IO | 실시간 양방향 통신 |
| 오디오 재생 | MediaSource Extensions | 끊김 없는 스트리밍 |

### 차별화 기술
1. **실시간 동시 번역**: 가이드 음성 → 텍스트 → 다국어 번역 (1초 이내 처리)
2. **지연 참여자 동기화**: 중간에 참여한 관광객도 즉시 오디오/자막 수신 가능
3. **모바일 최적화**: Android/iOS 브라우저에서 앱 설치 없이 즉시 작동
4. **자동 폴백**: WebRTC 연결 실패 시 WebSocket으로 자동 전환
5. **마이크 충돌 방지**: Android 기기에서 오디오 녹음과 음성인식 동시 작동

## 3. 사용자 구분 및 기능

### 가이드 (Speaker)
- 마이크를 통한 실시간 음성 방송
- 실시간 음성인식 텍스트 확인 (텔레프롬프터 기능)
- 방송 시작/정지 제어
- 장소 정보 등록/관리
- 세션 기록 다운로드 및 AI 요약

### 관광객 (Listener)
- 실시간 오디오 스트림 수신
- 100+ 언어 중 선택하여 번역 자막 수신
- TTS(음성합성) 기능으로 번역된 음성 청취
- 자동 연결 (별도 조작 불필요)

### 모니터 (Admin)
- 실시간 접속자 현황 대시보드
- 가이드 온라인/방송 상태 확인
- 언어별 관광객 분포 통계

## 4. 시스템 아키텍처

### 데이터 흐름도
```
┌─────────────────────────────────────────────────────────────────┐
│                        가이드 디바이스                            │
│  [마이크] → [음성 캡처] → [STT 음성인식] → [텍스트 추출]           │
└──────────────────────────────┬──────────────────────────────────┘
                               │ WebSocket/WebRTC
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│                          서버                                    │
│  [오디오 수신] → [번역 엔진 (100+ 언어)] → [브로드캐스트]          │
│  [초기화 세그먼트 캐싱] → [지연 참여자 동기화]                     │
└──────────────────────────────┬──────────────────────────────────┘
                               │ Socket.IO
              ┌────────────────┼────────────────┐
              ▼                ▼                ▼
    ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
    │ 관광객 A     │  │ 관광객 B     │  │ 관광객 N     │
    │ 영어 자막    │  │ 일본어 자막  │  │ 선택 언어    │
    │ + 오디오     │  │ + 오디오     │  │ + 오디오     │
    └──────────────┘  └──────────────┘  └──────────────┘
```

### 기술 스택
- **백엔드**: Python, FastAPI, Socket.IO
- **프론트엔드**: JavaScript, HTML5, CSS3
- **데이터베이스**: SQLite
- **실시간 통신**: WebRTC, WebSocket
- **번역**: deep-translator (Google Translate)

## 7. 특허 출원 포인트

### 주요 청구항

**청구항 1: 실시간 다국어 동시 번역 오디오 가이드 방법**
> 가이드의 음성을 실시간으로 수신하여 음성인식(STT)으로 텍스트로 변환하고, 변환된 텍스트를 복수의 언어로 동시에 번역하여 각 관광객에게 선택된 언어의 번역 자막과 원본 오디오를 동시에 전송하는 실시간 다국어 오디오 가이드 방법

**청구항 2: 지연 참여자 동기화 시스템**
> 오디오 스트리밍 중 초기화 세그먼트를 서버에 캐싱하고, 방송 중간에 참여한 사용자에게 캐싱된 초기화 세그먼트를 우선 전송하여 즉시 오디오 재생이 가능하도록 하는 지연 참여자 동기화 시스템

**청구항 3: 하이브리드 스트리밍 전송 방법**
> WebRTC를 우선적으로 시도하고, 연결 실패 시 WebSocket으로 자동 전환하여 다양한 네트워크 환경에서 안정적인 오디오 스트리밍을 제공하는 하이브리드 전송 방법

**청구항 4: 모바일 마이크 충돌 방지 방법**
> 모바일 기기에서 오디오 녹음(MediaRecorder)과 음성인식(SpeechRecognition)을 동시에 사용할 때 발생하는 마이크 충돌을 방지하기 위해 음성인식 시작을 지연시키고 자동 재시도하는 방법

**청구항 5: 웹 기반 실시간 오디오 가이드 시스템**
> 별도의 애플리케이션 설치 없이 웹 브라우저만으로 작동하며, 가이드의 음성을 실시간으로 100개 이상의 언어로 번역하여 관광객에게 전송하는 웹 기반 실시간 오디오 가이드 시스템
